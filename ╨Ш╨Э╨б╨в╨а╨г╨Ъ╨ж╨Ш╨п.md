# 🏭 Бот для цементного завода — Инструкция по запуску

## Что делает система

- **Дилер** пишет боту → создаёт заявку (номер машины + тонны)
- **Весовщик** меняет статус: Ожидание → Уехал → вводит реальный вес
- **Ты (админ)** видишь все заявки, статистику по дилерам, получаешь уведомления о расхождениях
- **Google Sheets** — вся история хранится там, можешь открыть в любой момент

---

## ШАГ 1 — Создать Telegram бота

1. Открой Telegram, найди **@BotFather**
2. Напиши `/newbot`
3. Придумай имя (пример: `Цементный завод`)
4. Придумай username (пример: `cement_zavod_bot`)
5. BotFather даст тебе **TOKEN** — сохрани его! Выглядит так:
   ```
   1234567890:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

---

## ШАГ 2 — Узнать свой Telegram ID

1. Напиши боту **@userinfobot** в Telegram
2. Он ответит твой ID, например: `123456789`
3. Сохрани этот номер — это твой **ADMIN_ID**

---

## ШАГ 3 — Настроить Google Sheets

### 3.1 — Создать таблицу
1. Открой [Google Sheets](https://sheets.google.com)
2. Создай новую таблицу, назови её "Цемент Завод"
3. Скопируй ID из URL:
   ```
   https://docs.google.com/spreadsheets/d/ВОТ_ЭТО_ID/edit
   ```

### 3.2 — Создать сервисный аккаунт
1. Открой [Google Cloud Console](https://console.cloud.google.com)
2. Создай новый проект (или выбери существующий)
3. Зайди в **APIs & Services → Library**
4. Найди и включи **Google Sheets API**
5. Найди и включи **Google Drive API**
6. Зайди в **APIs & Services → Credentials**
7. Нажми **Create Credentials → Service Account**
8. Имя: `cement-bot`, нажми Create
9. На следующем шаге роль не выбирай, просто Done
10. Кликни на созданный аккаунт → вкладка **Keys**
11. **Add Key → Create new key → JSON** → скачается файл `credentials.json`

### 3.3 — Дать доступ к таблице
1. Открой скачанный `credentials.json`
2. Найди строку `"client_email"` — скопируй email (похож на `cement-bot@project.iam.gserviceaccount.com`)
3. Открой свою Google таблицу
4. Нажми **Поделиться** (кнопка вверху справа)
5. Вставь этот email, дай роль **Редактор**, нажми Отправить

---

## ШАГ 4 — Запустить бота на Railway (бесплатно)

### 4.1 — Зарегистрироваться
1. Открой [railway.app](https://railway.app)
2. Войди через GitHub (нужен аккаунт на GitHub)

### 4.2 — Загрузить код на GitHub
1. Зайди на [github.com](https://github.com), создай новый репозиторий (назови `cement-bot`)
2. Загрузи все файлы из этой папки:
   - `bot.py`
   - `sheets.py`
   - `requirements.txt`
   - `Procfile`
   - `runtime.txt`

### 4.3 — Создать проект на Railway
1. На Railway нажми **New Project → Deploy from GitHub repo**
2. Выбери свой репозиторий `cement-bot`
3. Railway сам обнаружит Procfile и запустит

### 4.4 — Добавить переменные окружения
В Railway открой свой проект → вкладка **Variables**, добавь:

| Переменная | Значение |
|---|---|
| `BOT_TOKEN` | Токен от BotFather |
| `ADMIN_ID` | Твой Telegram ID |
| `WEIGHER_IDS` | Telegram ID весовщика (если несколько — через запятую: `111,222`) |
| `GOOGLE_SHEET_ID` | ID твоей Google таблицы |
| `GOOGLE_CREDS_JSON` | Содержимое файла credentials.json (скопируй весь текст целиком) |

---

## ШАГ 5 — Первый запуск

1. После добавления переменных Railway автоматически перезапустит бота
2. Открой своего бота в Telegram
3. Напиши `/start`
4. Ты увидишь меню администратора
5. Дай дилерам username бота — они тоже пишут `/start` и видят меню дилера

---

## Кто что видит

| Роль | Что видит в боте |
|---|---|
| **Дилер** (все остальные) | Создать заявку, посмотреть свои заявки |
| **Весовщик** (WEIGHER_IDS) | Менять статус машин, вводить реальный вес |
| **Ты (ADMIN_ID)** | Всё: статистика за день, по дилерам, расхождения, активные заявки + уведомления |

---

## Google Таблица — структура

Бот сам создаст два листа:

**Лист "Заявки":**
ID | Дата | Дилер | Dealer_ID | Номер машины | Заявлено (т) | Факт (т) | Расхождение (т) | Статус | Время

**Лист "Дилеры":**
Dealer_ID | Имя | Username | Дата регистрации

---

## Статусы заявки

🟡 **Ожидание** — заявка создана дилером, машина ещё на заводе
🚛 **Уехал** — весовщик отметил что машина ушла
✅ **Завершён** — реальный вес введён, расхождений нет
⚠️ **Расхождение** — реальный вес отличается от заявленного больше чем на 0.5 тонны → тебе приходит уведомление

---

## Если что-то не работает

- Проверь что все переменные окружения заданы правильно
- Убедись что сервисному аккаунту дан доступ к таблице
- В Railway можно смотреть логи: вкладка **Deployments → View Logs**
